üîÑ Windows Updates and Patch Management

üö® Critical Priority

Windows systems are frequently targeted. Enable automatic updates and regularly check for critical security patches from Microsoft.

# Check for Windows Updates Get-Module PSWindowsUpdate -ListAvailable Install-Module PSWindowsUpdate -Force Import-Module PSWindowsUpdate # Get available updates Get-WUList # Install all updates Install-WindowsUpdate -AcceptAll -AutoReboot # Check Windows Update service status Get-Service wuauserv # Enable automatic updates via registry Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" -Name "NoAutoUpdate" -Value 0 Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" -Name "AUOptions" -Value 4 # Check last boot time (reboot after updates) Get-CimInstance -ClassName Win32_OperatingSystem | Select LastBootUpTime
What this does: Ensures your Windows system has the latest security patches installed and configures automatic updates to prevent zero-day exploits.

üë§ User Account Control (UAC) and Account Security

‚ö†Ô∏è UAC Bypass Vulnerabilities

Weak UAC settings and poor user account management are common attack vectors. Proper UAC configuration can prevent 70% of privilege escalation attacks.

# Enable UAC at highest level Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" -Name "ConsentPromptBehaviorAdmin" -Value 2 Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" -Name "EnableLUA" -Value 1 # Disable built-in Administrator account Get-LocalUser Administrator | Disable-LocalUser # Create a new admin user $SecurePassword = ConvertTo-SecureString "ComplexPassword123!" -AsPlainText -Force New-LocalUser "SecAdmin" -Password $SecurePassword -Description "Security Administrator" Add-LocalGroupMember -Group "Administrators" -Member "SecAdmin" # Set password policy Set-LocalUser -Name "SecAdmin" -PasswordNeverExpires $false net accounts /maxpwage:90 /minpwage:1 /minpwlen:8 /uniquepw:5 # List all local users Get-LocalUser | Select Name, Enabled, LastLogon # Remove unnecessary users Remove-LocalUser -Name "Guest" # Check for empty passwords Get-LocalUser | Where-Object {$_.PasswordRequired -eq $false}

üí° Account Security Best Practices

  • Always run with standard user privileges
  • Use complex passwords (minimum 12 characters)
  • Enable account lockout policies
  • Regularly audit user accounts
  • Implement multi-factor authentication

üî• Windows Defender Firewall Configuration

# Enable Windows Defender Firewall for all profiles Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True # Set default actions Set-NetFirewallProfile -Profile Domain -DefaultInboundAction Block -DefaultOutboundAction Allow Set-NetFirewallProfile -Profile Public -DefaultInboundAction Block -DefaultOutboundAction Allow Set-NetFirewallProfile -Profile Private -DefaultInboundAction Block -DefaultOutboundAction Allow # Check firewall status Get-NetFirewallProfile | Select Name, Enabled, DefaultInboundAction, DefaultOutboundAction # Allow Remote Desktop (if needed) New-NetFirewallRule -DisplayName "Allow RDP" -Direction Inbound -Protocol TCP -LocalPort 3389 -Action Allow # Block specific ports New-NetFirewallRule -DisplayName "Block Telnet" -Direction Inbound -Protocol TCP -LocalPort 23 -Action Block # View active firewall rules Get-NetFirewallRule | Where-Object {$_.Enabled -eq 'True'} | Select DisplayName, Direction, Action
What this does: Configures Windows Defender Firewall with secure defaults and creates specific rules for essential services.
# Create firewall rule for specific IP ranges New-NetFirewallRule -DisplayName "Allow Corporate Network" -Direction Inbound -RemoteAddress 192.168.1.0/24 -Action Allow # Block all inbound connections except established New-NetFirewallRule -DisplayName "Block New Inbound" -Direction Inbound -Action Block -Profile Any # Enable logging Set-NetFirewallProfile -Profile Domain,Public,Private -LogAllowed True -LogBlocked True -LogMaxSizeKilobytes 4096 # Monitor firewall logs Get-Content "C:\Windows\System32\LogFiles\Firewall\pfirewall.log" -Tail 10 # Create application-specific rules New-NetFirewallRule -DisplayName "Block Calculator" -Direction Outbound -Program "C:\Windows\System32\calc.exe" -Action Block
What this does: Creates advanced firewall rules for network segmentation, logging, and application-level controls.

üîê Registry Security Hardening

‚ö†Ô∏è Registry Vulnerabilities

The Windows Registry contains critical system settings. Improper configurations can lead to privilege escalation and system compromise.

# Disable SMBv1 (prevents WannaCry-style attacks) Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters" -Name "SMB1" -Value 0 # Disable LLMNR (prevents LLMNR poisoning) New-Item -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient" -Force Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient" -Name "EnableMulticast" -Value 0 # Disable NetBIOS over TCP/IP $adapters = Get-WmiObject -Class Win32_NetworkAdapterConfiguration | Where-Object {$_.IPEnabled -eq $true} foreach ($adapter in $adapters) { $adapter.SetTcpipNetbios(2) } # Enable LSA Protection Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name "RunAsPPL" -Value 1 # Disable WDigest authentication Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest" -Name "UseLogonCredential" -Value 0 # Enable Credential Guard (Windows 10 Enterprise) Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\DeviceGuard" -Name "EnableVirtualizationBasedSecurity" -Value 1 Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\DeviceGuard" -Name "RequirePlatformSecurityFeatures" -Value 1
What this does: Modifies critical registry settings to disable vulnerable protocols, enable security features, and protect against common attack vectors.

üìã Windows Event Logging and Monitoring

# Enable PowerShell logging Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ModuleLogging" -Name "EnableModuleLogging" -Value 1 Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging" -Name "EnableScriptBlockLogging" -Value 1 # Configure security log size wevtutil sl Security /ms:1073741824 # Enable process creation logging Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Audit" -Name "ProcessCreationIncludeCmdLine_Enabled" -Value 1 # Check for failed login attempts Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} -MaxEvents 10 | Select TimeCreated, Message # Monitor privilege escalation attempts Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4672} -MaxEvents 5 # Check for new service installations Get-WinEvent -FilterHashtable @{LogName='System'; ID=7045} -MaxEvents 10 # Export security logs for analysis wevtutil epl Security C:\SecurityLogs\Security_$(Get-Date -Format 'yyyyMMdd').evtx # Set up log forwarding (to SIEM) winrm quickconfig -force wecutil cs subscription.xml

üí° Key Events to Monitor

  • 4624/4625: Successful/Failed logons
  • 4672: Special privileges assigned
  • 4697: Service installed
  • 4698/4699: Scheduled task created/deleted
  • 4103/4104: PowerShell execution
  • 1: Process creation (Sysmon)

üö´ Disable Unnecessary Services and Features

# Disable unnecessary services $ServicesToDisable = @( 'Fax', 'TelnetClnt', 'TFTP', 'RemoteRegistry', 'RemoteAccess', 'TermService' ) foreach ($service in $ServicesToDisable) { Stop-Service -Name $service -Force -ErrorAction SilentlyContinue Set-Service -Name $service -StartupType Disabled -ErrorAction SilentlyContinue } # Remove unnecessary Windows features Disable-WindowsOptionalFeature -Online -FeatureName "TelnetClient" -NoRestart Disable-WindowsOptionalFeature -Online -FeatureName "TFTP" -NoRestart Disable-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol" -NoRestart # List all running services Get-Service | Where-Object {$_.Status -eq 'Running'} | Select Name, DisplayName, StartType # Check installed Windows features Get-WindowsOptionalFeature -Online | Where-Object {$_.State -eq 'Enabled'} | Select FeatureName

‚ö†Ô∏è Service Hardening Priority

Unnecessary services increase attack surface. Disable services you don't need, but be careful not to disable critical system services.

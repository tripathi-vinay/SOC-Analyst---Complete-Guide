üîê System Updates and Package Management

üö® Critical Priority

Keep your system updated! 90% of successful attacks exploit known vulnerabilities that have available patches.

# Update package lists sudo apt update # Upgrade all packages sudo apt upgrade -y # Remove unused packages sudo apt autoremove -y # Check for security updates sudo unattended-upgrade --dry-run # Enable automatic security updates sudo dpkg-reconfigure -plow unattended-upgrades
What this does: Updates your system with the latest security patches and removes unnecessary packages that could be potential attack vectors.
# Update all packages (RHEL/CentOS 8+) sudo dnf update -y # For older versions (CentOS 7) sudo yum update -y # Check for security updates only sudo dnf updateinfo list security # Install security updates only sudo dnf update --security -y
What this does: Updates Red Hat based systems with security patches and provides options for security-only updates.
# Update system sudo pacman -Syu # Check for orphaned packages pacman -Qdt # Remove orphaned packages sudo pacman -Rs $(pacman -Qtdq)
What this does: Updates Arch Linux system and removes orphaned packages that are no longer needed.

üë§ User Account Security

‚ö†Ô∏è Common Vulnerability: Weak User Management

Default accounts, weak passwords, and excessive privileges are common attack vectors. Always follow the principle of least privilege.

# Create a new user with limited privileges sudo useradd -m -s /bin/bash -G sudo username # Set strong password policy sudo passwd username # Lock unused accounts sudo usermod -L nobody sudo usermod -L games # Check for accounts with empty passwords sudo awk -F: '($2 == "") {print}' /etc/shadow # Remove unnecessary users sudo userdel -r olduser # Set password aging policy sudo chage -M 90 -m 7 -W 7 username # View current password policies sudo chage -l username

üí° Best Practices for User Management

  • Use strong passwords (minimum 12 characters)
  • Enable two-factor authentication when possible
  • Regularly audit user accounts and remove unused ones
  • Implement password rotation policies
  • Use sudo instead of root login

üî• Firewall Configuration (UFW & iptables)

# Enable UFW firewall sudo ufw enable # Set default policies sudo ufw default deny incoming sudo ufw default allow outgoing # Allow SSH (change port if needed) sudo ufw allow 22/tcp # Allow HTTP and HTTPS sudo ufw allow 80/tcp sudo ufw allow 443/tcp # Allow specific IP for SSH sudo ufw allow from 192.168.1.100 to any port 22 # View firewall status sudo ufw status verbose # View numbered rules sudo ufw status numbered # Delete a rule sudo ufw delete 2
What this does: Configures a basic firewall using UFW (Uncomplicated Firewall) to block unnecessary incoming connections while allowing essential services.
# Flush existing rules sudo iptables -F sudo iptables -X sudo iptables -t nat -F sudo iptables -t nat -X # Set default policies sudo iptables -P INPUT DROP sudo iptables -P FORWARD DROP sudo iptables -P OUTPUT ACCEPT # Allow loopback sudo iptables -I INPUT 1 -i lo -j ACCEPT # Allow established connections sudo iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT # Allow SSH sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT # Block common attack patterns sudo iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP sudo iptables -A INPUT -p tcp --tcp-flags ALL ALL -j DROP # Save rules (Ubuntu/Debian) sudo iptables-save > /etc/iptables/rules.v4
What this does: Advanced firewall configuration using iptables for more granular control over network traffic and blocking common attack patterns.

üîë SSH Hardening

‚ö†Ô∏è SSH Vulnerabilities

SSH is often targeted by attackers. Default configurations are often insecure. Proper SSH hardening can prevent 80% of brute force attacks.

# Backup original SSH config sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup # Edit SSH configuration sudo nano /etc/ssh/sshd_config # Key configurations to add/modify: # Change default port Port 2222 # Disable root login PermitRootLogin no # Disable password authentication (use keys only) PasswordAuthentication no PubkeyAuthentication yes # Limit user access AllowUsers yourusername # Set login grace time LoginGraceTime 60 # Limit max auth tries MaxAuthTries 3 # Disable empty passwords PermitEmptyPasswords no # Disable X11 forwarding if not needed X11Forwarding no # Generate SSH key pair (client side) ssh-keygen -t rsa -b 4096 -C "your_email@example.com" # Copy public key to server ssh-copy-id -p 2222 username@server_ip # Restart SSH service sudo systemctl restart sshd # Check SSH service status sudo systemctl status sshd

üí° SSH Security Checklist

  • Change default SSH port from 22
  • Disable root login completely
  • Use SSH keys instead of passwords
  • Implement fail2ban for brute force protection
  • Use strong ciphers and disable weak ones
  • Regularly monitor SSH logs

üìÅ File System Security

# Find files with SUID bit set find / -type f -perm -4000 2>/dev/null # Find world-writable files find / -type f -perm -002 2>/dev/null # Find files without owner find / -nouser -o -nogroup 2>/dev/null # Secure important directories sudo chmod 700 /root sudo chmod 700 /home/* sudo chmod 644 /etc/passwd sudo chmod 600 /etc/shadow # Set up file integrity monitoring sudo apt install aide sudo aideinit sudo cp /var/lib/aide/aide.db.new /var/lib/aide/aide.db # Check file integrity sudo aide --check # Enable filesystem monitoring with auditd sudo apt install auditd sudo auditctl -w /etc/passwd -p wa -k passwd_changes sudo auditctl -w /etc/shadow -p wa -k shadow_changes sudo auditctl -w /etc/group -p wa -k group_changes

üí° File System Hardening Tips

  • Regularly scan for SUID/SGID files
  • Monitor critical system files for changes
  • Use proper file permissions (principle of least privilege)
  • Enable file system encryption for sensitive data
  • Implement access controls with SELinux/AppArmor

üõ°Ô∏è Network Security

# Disable IP forwarding echo 'net.ipv4.ip_forward = 0' | sudo tee -a /etc/sysctl.conf # Disable ICMP redirects echo 'net.ipv4.conf.all.accept_redirects = 0' | sudo tee -a /etc/sysctl.conf echo 'net.ipv6.conf.all.accept_redirects = 0' | sudo tee -a /etc/sysctl.conf # Enable SYN flood protection echo 'net.ipv4.tcp_syncookies = 1' | sudo tee -a /etc/sysctl.conf # Ignore ping requests echo 'net.ipv4.icmp_echo_ignore_all = 1' | sudo tee -a /etc/sysctl.conf # Apply sysctl changes sudo sysctl -p # Install and configure fail2ban sudo apt install fail2ban # Create fail2ban local config sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local # Edit fail2ban configuration sudo nano /etc/fail2ban/jail.local # Start fail2ban service sudo systemctl enable fail2ban sudo systemctl start fail2ban # Check fail2ban status sudo fail2ban-client status sudo fail2ban-client status sshd

‚ö†Ô∏è Network Attack Vectors

Common network attacks include DDoS, man-in-the-middle, and port scanning. Proper network hardening can mitigate these risks significantly.

  • Implement log retention policies
